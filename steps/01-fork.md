## What Is a Repository?

A **repository** (or "repo" for short) is just a folder that git is tracking. Every file you add, every change you make — git keeps a record of it all so you can look back at any point in history.

This curriculum lives in a repository on GitHub. Your job right now is to make your own personal copy of it.

---

## What Is a Fork?

Think of the original repository like a textbook in the school library. Everyone can read it, but you cannot write in it. **Forking** is like getting your own personal copy of that textbook — one that belongs to you, that you can write all over, and that will not affect anyone else's copy.

When you fork, GitHub creates a full copy of the repository under your own account. It looks identical to the original, but it is yours.

---

## Your Task: Fork This Repo

### Using the GitHub Website

1. Make sure you are logged into GitHub.
2. At the top of this page (or the page you came from), look for the **Fork** button in the upper-right corner of the page, next to the Star and Watch buttons.
3. Click **Fork**.
4. GitHub will ask where to fork to — select your own username.
5. Wait a few seconds while GitHub copies the repository.

That is it. You are now looking at your fork.

---

## How to Know You Did It

Look at your browser's address bar. It should show:

```
https://github.com/YOUR-USERNAME/tech-journey-github
```

If you see your username there instead of someone else's, you forked successfully.
